# sitem-i-notes

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态文件的形式存储，可通过 GitHub Pages 直接访问。

## 项目简介

sitem-i-notes 是一个轻量级的 HTML 页面归档仓库。它不针对任何特定域名或网站，而是作为通用存放空间，用于整理和展示各种独立的网页内容。适合存放笔记、工具页面、资料备份、临时演示等场景。

## 目录说明

```
/
├── index.html          # 仓库首页（可选）
├── pages/              # 页面归档目录
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
└── assets/             # 公共资源（CSS、JS、图片等）
    ├── style.css
    └── script.js
```

- `pages/` 目录存放所有独立的 HTML 页面，每个文件为一个独立页面。
- `assets/` 目录存放页面引用的公共资源文件，如图片、样式表、脚本等。
- 根目录下的 `index.html` 可作为导航页，非必须。

## 页面归档说明

- 每个 HTML 页面应保持独立完整，避免跨文件依赖（除非引用 `assets/` 中的公共资源）。
- 页面命名建议使用英文小写，单词间用连字符分隔，例如 `my-note.html`。
- 归档内容不涉及具体域名或商业推广，仅供技术存档与展示使用。

## 维护说明

- 如需添加新页面，请在 `pages/` 目录下创建 HTML 文件，并更新 `index.html` 中的导航链接（如有）。
- 修改公共资源时请注意保持向后兼容，避免影响已归档页面。
- 本仓库不提供自动部署脚本，建议手动提交更新后等待 GitHub Pages 自动构建。
